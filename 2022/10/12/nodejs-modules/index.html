1. 模块化概述

1.1 什么是模块化

模块化是将代码分割成独立的模块，每个模块负责特定的功能或逻辑。这种编程范式将大型程序分解为相互依赖的小文件，提高了代码的可维护性、可重用性和可测试性。

模块化的核心优势：

  1. 可维护性：每个模块独立存在，修改一个模块不会影响其他模块
  2. 可重用性：模块可以在不同项目中复用，减少重复代码
  3. 可测试性：模块可以单独测试，确保功能正确性
  4. 命名空间管理：避免全局变量污染，减少命名冲突

1.2 模块化的历史背景

JavaScript 在浏览器端的模块化方式经历了从全局作用域到基于 script 标签的引用再到现代化的模块系统（如 ES6 模块）。然而，Node.js 的模块化是基于 CommonJS 规范开发的，这使得 Node.js 在服务器端也能够非常方便地实现模块化。

1.3 模块化解决的问题

  1. 命名冲突：传统JS开发中全局变量容易冲突，模块化提供了独立作用域
  2. 文件依赖：明确模块间的依赖关系，避免手动管理script标签顺序
  3. 代码组织：使项目结构更清晰，便于团队协作开发

3. ES Modules 简介

3.1 基本概念

ES Modules (ESM) 是 JavaScript 的官方模块标准（ECMAScript 2015 引入），用于替代传统的 CommonJS（Node.js 原生的 require 语法）。

  • 设计目标：实现静态化模块依赖，支持异步加载，适合浏览器和服务器端。
  • 核心优势：
    • 静态分析（编译时确定依赖关系，利于优化）。
    • 原生浏览器支持（无需打包工具即可直接使用）。
    • 异步加载（更适合现代应用）。

3.2 基本用法

4. CommonJS VS ES Modules

在 Node.js 中，require() 和 import 是两种不同的模块系统（CommonJS 和 ES Modules），它们的加载机制有本质区别。以下是两个模块系统的比较：

4.1 require() 是同步的，无法直接加载 ES 模块

  • CommonJS 的 require()

    • 是同步阻塞的（模块加载完成后才会执行后续代码）。
    • 只能加载 .js、.json、.node 等文件，默认不支持 .mjs（ES 模块）。
    • 如果直接 require('./es-module.mjs') 会报错：

  • ES Modules 的 import

    • 是异步的（底层使用 Promise）。
    • 必须用于 .mjs 文件或 package.json 中声明 "type": "module" 的 .js 文件。

4.2 在 CommonJS 中加载 ES 模块的方法

由于 require() 无法直接加载 ES 模块，Node.js 提供了两种解决方案：

方法 1：动态 import() 函数（推荐）

  • import() 是 ES 模块的异步加载语法，返回一个 Promise，可在 CommonJS 中使用。

5. Node.js核心模块

5.1 global对象

Node.js中的全局对象，用来存储全局变量：

  • global：Node.js全局对象（类似浏览器的window）
  • globalThis：ES标准全局对象名称，Node.js中等于global

与浏览器 window 对象的区别

5.3 process模块

process 是 Node.js 的一个核心全局对象，提供了与当前 Node.js 进程交互的多种功能和方法。它不需要通过 require() 导入即可使用。

process 常用方法和属性

分类 属性/方法 类型 说明 示例
进程信息 process.pid 属性 当前进程的 PID console.log(process.pid); // 12345
process.platform 属性 操作系统平台 ('win32', 'linux', 'darwin') if (process.platform === 'win32') { ... }
process.arch 属性 CPU 架构 ('x64', 'arm') console.log(process.arch); // 'x64'
命令行参数 process.argv 属性（数组） 命令行参数（[node路径, 脚本路径, ...args]） node app.js arg1 → process.argv[2] === 'arg1'
环境变量 process.env 属性（对象） 环境变量对象 console.log(process.env.NODE_ENV); // 'development'
进程控制 process.exit([code]) 方法 退出进程（默认 code=0，非 0 表示失败） if (err) process.exit(1);

5.6 HTTP模块

用于创建 HTTP 服务和客户端请求，处理 Web 服务器相关的操作。

常用方法：

  • http.createServer(callback)：创建一个 HTTP 服务器

  • request.url：获取请求的 URL（不包括域名）
  • request.headers：获取请求头（对象形式）
  • request.on('data', callback)： 接收请求体数据（POST/PUT）
  • request.on('end', callback)：请求体接收完毕
  • request.pipe(destination) ：将请求数据流式传输到其他流
  • response.setHeader(name, value)：设置单个响应头

  • response.write(data)：向客户端响应数据

6. 总结

在 Node.js 的生态系统中，模块系统经历了从 CommonJS 到 ES Modules 的演进，每种方式都有其独特的特点和适用场景。

特性 ES Modules CommonJS
语法 import/export require/module.exports
加载方式 静态（编译时分析） 动态（运行时加载）
浏览器支持 原生支持 需打包工具（如 Webpack）
Node.js 支持 .mjs 或 "type": "module" 默认支持 .js
顶层 await 支持 不支持
适用场景 浏览器环境和静态优化 服务端文件系统 I/O

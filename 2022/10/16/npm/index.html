项目初始化与依赖管理

初始化项目

  • npm init：创建 package.json 文件

    • 该命令会根据交互式提示，生成一个基本的 package.json 文件。

  • npm init -y：快速生成 package.json 文件

    • 这个命令会自动接受所有默认配置，并快速生成 package.json 文件，省去交互式输入。

安装依赖

  • npm install <package-name> --save：安装生产依赖

    • 默认会安装最新版本的包，并将其记录在 package.json 的 dependencies 字段下。

  • npm install <package-name>@<version>：安装指定版本的依赖

  • npm install <package-name> --save-dev：安装开发依赖

  • npm install -g <package-name>：安装全局工具

  • npm install：安装项目中 package.json 中列出的所有依赖

卸载依赖

  • npm uninstall <package-name>：卸载指定的包

  • npm uninstall <package-name> --save-dev：从 devDependencies 中卸载依赖

更新依赖

  • npm update：更新所有依赖包到符合 package.json 中指定的版本范围

  • npm update <package-name>：更新指定包

查看包信息

  • npm list：列出当前项目中所有安装的依赖

  • npm list <package-name>：查看指定包及其依赖的版本

  • npm outdated：查看项目中哪些包有新版本可用

  • npm audit：检查安全漏洞

package.json 常见配置

项目元信息

  • name：项目名称
  • version：项目版本
  • description：项目描述
  • main：项目的入口文件（如 index.js）

依赖配置

  • dependencies：生产环境依赖项，项目运行时所需的包
  • devDependencies：开发环境依赖项，项目开发时所需的包

版本控制符号

符号 示例 说明
^ ^1.2.3 允许更新次版本和修订号（1.x.x）
~ ~1.2.3 只允许更新修订号（1.2.x）
> >1.2.3 大于指定版本
= =1.2.3 精确版本
- 1.2.3 - 2.3.4 版本范围

脚本配置

package.json 中的 scripts 字段定义了常用的命令脚本，使用 npm run <script-name> 来执行这些脚本。

  • start：通常用于启动项目
  • test：用于运行测试
  • build：用于构建项目

项目配置

  • engines：指定 Node.js 或 npm 的版本要求
  • license：项目的许可证

环境变量配置

  • scripts 中使用环境变量：在 scripts 部分中使用环境变量来区分不同的环境，如开发环境、生产环境等。

浏览器兼容性配置

npm 配置文件

除了 package.json 外，npm 还使用 .npmrc 配置文件来管理 npm 的配置选项。

常见配置：

  • registry：设置 npm 的镜像源
    • 默认是 https://registry.npmjs.org/
    • 可以设置为国内镜像源，如 https://registry.npm.taobao.org/
  • prefix：指定全局安装包的位置

一、Docker 是什么？为什么需要它？

在软件开发中，我们经常遇到这样的困境：“在我的机器上可以运行，为什么到了你的机器上就报错了？”这个问题通常是由于环境不一致导致的：操作系统、依赖库、配置文件等细微差别都可能让程序“水土不服”。

Docker 就是为了解决这个问题而生的。

Docker 是一个开源的容器化平台，它允许你将应用程序及其所有依赖项（代码、运行时、系统工具、系统库、设置）打包成一个标准化、轻量级、可移植的容器。这个容器可以在任何安装了 Docker 的机器上运行，保证环境完全一致。

Docker 与虚拟机的区别

特性 虚拟机 (VM) Docker 容器
抽象层级 硬件级别 操作系统级别
隔离性 完全隔离，更安全 进程级别隔离，相对轻量
启动速度 慢 (分钟级) 快 (秒级)
性能 有损耗，占用资源多 接近原生，占用资源少
磁盘占用 GB 级别 MB 级别
本质 一套完整的虚拟化系统 一个隔离的进程

简单来说，虚拟机是虚拟出一整套硬件，并在上面运行一个完整的操作系统。而 Docker 容器是直接在你的主机操作系统上隔离运行进程，共享主机内核，因此极其轻量和高效。

二、核心概念

要理解 Docker，必须掌握三个核心概念：镜像、容器、仓库。它们的关系可以用面向对象编程来类比：

1. 镜像 (Image)

  • 类比： 面向对象中的 类 (Class)
  • 是什么： 一个只读的模板。它包含了创建 Docker 容器所需的所有文件和配置信息（如代码、运行时环境、库、环境变量、配置文件等）。镜像本身是静态的、不可改变的。
  • 例子： 一个 Ubuntu 镜像、一个安装了 Nginx 的镜像、一个包含了你的 Python 应用的镜像。

2. 容器 (Container)

  • 类比： 面向对象中 类的实例 (Instance)
  • 是什么： 镜像的运行实例。当你运行一个镜像时，Docker 会在镜像的上方创建一个可写层，这个运行中的实体就是容器。容器可以被启动、开始、停止、删除。每个容器都是相互隔离的。
  • 例子： 你基于 Ubuntu 镜像运行了一个容器，就像你安装并启动了一个 Ubuntu 系统一样，可以在里面执行命令。

3. 仓库 (Repository)

  • 类比： Git 仓库 或 应用商店
  • 是什么： 用来存放镜像的地方。Docker 官方提供了一个巨大的公共仓库 —— Docker Hub，就像 GitHub 一样，你可以在上面下载别人制作好的镜像，也可以上传自己的镜像。
  • 标签 (Tag)： 一个仓库里可以有多个镜像，通常用标签来区分不同版本（如 ubuntu:20.04, ubuntu:22.04）。

三者关系总结：
仓库 里存放着很多 镜像。你用 docker run 命令把 镜像 运行起来，就创建了一个 容器。

四、基本使用命令

1. 生命周期管理

  • docker run [OPTIONS] IMAGE [COMMAND]

    • 核心命令！ 从镜像创建并启动一个容器。
    • 常用选项:
      • -d: 后台运行容器（守护模式）
      • -it: 交互模式运行容器（通常一起使用）
      • --name: 为容器指定一个名字
      • -p: 端口映射，格式 主机端口:容器端口
      • -v: 数据卷映射，用于持久化数据
      • -e: 设置环境变量
    • 例子:

  • docker start/stop/restart CONTAINER

  • 启动 / 停止 / 重启一个已存在的容器

  • docker rm CONTAINER

    • 删除一个已停止的容器
    • -f: 强制删除（包括正在运行的容器）
    • docker rm -f $(docker ps -aq): 危险命令！ 强制删除所有容器

  • docker exec [OPTIONS] CONTAINER COMMAND

    • 在正在运行的容器中执行命令
    • 例子：

2. 镜像管理

  • docker pull IMAGE[:TAG]

    • 从仓库下载镜像到本地
    • docker pull ubuntu:20.04

  • docker images

    • 列出本地所有镜像

  • docker rmi IMAGE

    • 删除本地镜像

  • docker build [OPTIONS] PATH

    • 根据 Dockerfile 构建新镜像
    • -t: 给镜像打标签
    • docker build -t my-app:1.0 .

3. 信息查看

  • docker ps [OPTIONS]

    • 列出容器
    • -a: 列出所有容器（包括已停止的）

  • docker logs CONTAINER

    • 查看容器日志
    • -f: 实时跟踪日志

五、Dockerfile 实战

镜像是通过 Dockerfile 构建出来的。Dockerfile 是一个文本文件，包含了一系列指令，告诉 Docker 如何构建镜像。

一个简单的 Dockerfile 示例

常用 Dockerfile 指令

以下是最关键和最常用的指令：

指令 用途 示例
FROM 指定基础镜像。所有 Dockerfile 都必须以 FROM 开头（ARG 除外）。 FROM ubuntu:20.04
RUN 执行命令并创建新的镜像层。常用于安装软件包。 RUN apt-get update && apt-get install -y nginx
COPY 从构建上下文复制文件或目录到镜像内的路径。 COPY ./app /usr/src/app
ADD 类似 COPY，但功能更多（如自动解压 tar 包，从 URL 下载）。推荐优先使用 COPY。 ADD https://example.com/file.tar.gz /tmp/
WORKDIR 设置工作目录。相当于 cd，后续的 RUN, CMD, ENTRYPOINT, COPY, ADD 指令都会在此目录下执行。 WORKDIR /app
EXPOSE 声明容器运行时监听的端口。这只是一个文档化的提示，实际映射端口需要在 docker run 时用 -p 参数指定。 EXPOSE 80
ENV 设置环境变量。这个变量在构建阶段和容器运行时都存在。 ENV NODE_ENV production
ARG 定义构建时的变量，只在构建阶段有效，容器运行时不存在。 ARG APP_VERSION=1.0
CMD 指定容器启动时默认执行的命令。一个 Dockerfile 只能有一条 CMD 指令。如果被 docker run 后面的命令覆盖，则不会执行。 CMD ["nginx", "-g", "daemon off;"]
ENTRYPOINT 配置容器启动时运行的命令，并且不会被 docker run 后面的命令覆盖，而是会将其后面的参数作为参数传递给 ENTRYPOINT 指令指定的程序。 ENTRYPOINT ["echo"]
USER 指定运行容器时的用户名或 UID，后续的 RUN, CMD, ENTRYPOINT 都会以此身份运行。 USER nobody
VOLUME 创建一个匿名数据卷挂载点，用于存储动态数据。 VOLUME /var/log

六、典型工作流程

  1. 编写应用代码：开发你的应用程序
  2. 编写 Dockerfile：定义如何为应用构建镜像
  3. 构建镜像：使用 docker build -t my-app . 命令
  4. 运行容器：使用 docker run -d -p 4000:80 my-app 命令
  5. 测试访问：在浏览器中打开应用验证
  6. 上传分享：使用 docker push 将镜像上传到仓库
